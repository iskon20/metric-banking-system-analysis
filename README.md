## 4. Project Title  
Metric - Smart Banking Analytics System  

---

## 5. Problem Statement  

Banks generate large volumes of transaction and customer data daily.  
However, without a structured system, this data is often difficult to analyze effectively.  

This leads to:  

- Slow reporting processes  
- Limited understanding of customer behavior  
- Difficulty identifying valuable or risky clients  
- Delays in decision-making  

The Metric project aims to design a structured banking analytics system that improves data organization and supports better management decisions.

---

## 6. Project Objectives  

1. Define functional and non-functional requirements.  
2. Analyze banking workflows.  
3. Design UML system diagrams.  
4. Plan the database structure.  
5. Create dashboard concepts and documentation.  

---

## 7. Scope of the System  

The system will include:

- Role-based user access  
- Transaction monitoring  
- Customer segmentation  
- Analytical dashboards and reports  

The project focuses on system planning and modeling rather than full software implementation.
